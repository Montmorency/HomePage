Environments

In order to liberate neurons that would be better served soaking in malted hops I am collecting some useful links and notes for configuring your environment on UNIX and MacOS systems.

Determining Your Version of Linux

To determine what version of linux and the kernel is in use: 

$cat /etc/issue
  Scientific Linux release 6.6 (Carbon)
  Kernel \r on an \m
$uname -r
  2.6.32-431.17.1.el6.x86_64
$uname -i
  x86_64

Text Editors

For editing text files and developement in general I like to use VIM. I typically use the plainest vanilla configuration of vi available; typically only reconfiguring the default tab style to two white spaces. Here is a useful article for getting your environment set up with more bells and whistles: VIM Environment Configuration. Here are the core lines I like to add to ~/.vimrc whenever I'm on a new computer. 


syntax on
au BufNewFile, BufRead *.py
  set tabstop=2
  set softtabstop=2
  set shiftwidth=2
  set textwidth=140
  set expandtab
  set autoindent
  set fileformat=unix
  set ruler

SSH

In your ~/.ssh/config file it is a good idea to set up Host aliases. This makes using rsync and scp between machines a lot simpler. The typical pattern is below. 


Host *
ServerAliveInterval 240

Host nameofcomputer
Hostname comp.at.another.location.uk
User myusername
Port 22

General configurations can use pattern matching i.e. Host * to set configurations across every server we plan on connecting to. Here we are specifying how often to ping the remote server, ServerAliveInterval 240 , so that the connection stays alive (every 4 minutes in this case). By default ssh processes listen on port 22. On OS X El Capitan (v 10.11.6) my ssh_config is located in /etc/ssh/ssh_config. If you want to change the default port that ssh listens on the best way is probably to edit the /System/Library/LaunchDaemons/ssh.plist file as described here . Make sure you then unload and load the plist with launchctl or restart the machine.

Should you be consistently accessing a server and getting a message like "WARNING: REMOTE HOSE IDENTIFICATION HAS CHANGED!", and you are growing weary of that message, you can append the line: 


StrictHostKeyChecking no
to the relevant Host configuration block in your ~/.ssh/config file.

If you install Xquartz via Homebrew on Mac Os Sierra the plist file is located here: 

/Library/LaunchAgents/org.macosforge.xquartz.startx.plist
/Library/LaunchDaemons/org.macosforge.xquartz.startx.plist
The ssh config files are stored in /etc/ssh and the X11 files are stored in /opt/local/X11/ with a symlink to that location made in /usr/X11. Homebrew also creates two plist files in /Library/LaunchAgents/ and /Library/LaunchDaemons. You may need to modify the ssh config files to allow X11 forwarding. Also note that if you used to have macports installed and have since switched to homebrew there maybe some old macport plist files causing problems so it is best to remove these.

System Integrity Protection

If something ends up in /usr/ directory that shouldn't be there you may encounter a Operation Not Permitted error when trying to remove it. This is due to the system integrity protection feature on recent versions of macos. To disable this you can reboot in safe mode: by holding ⌘+R after the mac reboot chime. Open a terminal from the Utilities menu and disable system protection with 
$crsutil disable
$reboot

Git

I typically use Git for all revision/versioning needs. There are a couple key patterns I repeat with Git and are useful to remember. The git documentation is very good, and this interactive Git manual is also quite useful. Through the web interface for your git profile under settings you can add ssh keys to all the machines where you would like to work on different repos. You can clone a git repo using https or ssh. After cloning you may want to set some general configurations like your email and username. 


$git config --user.email myemail@emailserver.com
$git config --user.name  GitHubName
If you use ssh initially and later wish to switch to https you can check the URL of the repository with: 

$git remote -v
  origin https://github.com/MyGitProfile/MyRepo.git (fetch)
  origin https://github.com/MyGitProfile/MyRepo.git (push)
You can then tell git to use ssh instead with: 

$git remote set-url origin git@github.com:MyUserName/MyRepo.git
If you've set your public-key (id_rsa.pub) in github settings correctly you can fetch and push with impunity.

To create a new local branch: 


$git branch NewBranch
To list available local branches: 

$git branch
  NewBranch
* master
To switch so that future work will be done on NewBranch. 

$git checkout NewBranch
  On branch NewBranch
$git branch
* NewBranch
  master
You can now work away at whatever it is you want to do in your new branch: make the changes, add them, commit them, and push them. If this is your first time on the NewBranch when you commit you will be prompted to set a new place on your repo for the NewBranch to be stored. 

$git push --set-upstream origin NewBranch
After doing that we may decide NewBranch has no future. It is a dead end. Let's get rid of it. 

$git branch -d NewBranch
... and it's gone.

If you have forked a branch from another repository you can set upstream and rebase your fork following the instructions here .

Amazon web services

If you have an EC2 instance with amazon web services you can ssh into your node using the private key issued when you launch the node: 

$ssh -i "myprivatekey.pem" ubuntu@ec2-00-00-00-00.eu-west-1.compute.amazonaws.com
This website is deployed using Flask with an Apache web server. To start the apache server: 
$sudo systemctl start apache2
If I modify any of the pages or reconfigure the back end the service can be relaunched with: 

$sudo service apache2 restart

wsgi Configuration

On an ubuntu server the default welcome page for Apache (/var/www/html/index.html) gives details on the default configuration for the Apache2 server on Ubuntu. For details on configuring the permissions of the /var/www/ directory see the amazon documentation. The wsgi configuration file is named (website.wsgi), and is located in /var/www/website/. I needed to add the python modules, and the website module explicitly to the system path in this small wsgi script. This layout should be somewhat typical for most wsgi scripts: 

import sys
import logging
logging.basicConfig(stream=sys.stderr)

sys.path.append("/home/ubuntu/locationof/pythonX.X/site-packages")
sys.path.append("/home/ubuntu/pymodules/website")

from website import app as application
Further configuration of the server takes place in /etc/apache2/. Placing a website.conf file in /etc/apache2/sites-available/ should be sufficient to get things up and running. The configuration should include your domain name and directives to run the WSGI Daemon in the .conf file. 

< VirtualHost *:80 >
    ServerName website_name.co.uk

    WSGIDaemonProcess website threads=5
    WSGIScriptAlias / /var/www/website/website.wsgi

    < Directory /var/www/website >
        WSGIProcessGroup website
        WSGIApplicationGroup %{GLOBAL}
        WSGIScriptReloading On
        Order allow,deny
        Allow from all
        Deny from XXX.XXX.XXX.XXX
    </Directory>
</VirtualHost>
Make sure you have the wsgi module loaded. sudo apt-get install libapache2-mod-wsgi . Run sudo a2ensite website.conf to register the conf file with the apache server. To track requests being issued check /var/log/apache2/ .

Denying IP Adresses on apache server

If you find IP addresses doing naughty things in your access log the .conf file above shows how to match against restricted ips. VPN and Networking

To login to the local network via a portal I am using BIG-IP Edge Client. Generally to check the ip address assigned to your laptop ifconfig and nslookup are your friends: 


$ifconfig
  ...
  ppp0: flags=8051 mtu 1500
  inet 123.00.000.000 --> 137.00.000.0 netmask 0xffffffff 
$nslookup 123.00.000.000
  Server:   000.00.000.00
  Address:  000.00.000.00#00

  000.00.000.000.in-addr.arpa name = name.firepass.inst.ac.uk.
If you enable remote login in System Preferences/Sharing (on a mac) you can now access your device with ssh username@name.firepass.inst.ac.uk.
